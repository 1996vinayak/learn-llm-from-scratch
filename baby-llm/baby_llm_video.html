STEP 2

🔤 Tokenizer — The Code

Two simple operations: Encode & Decode

class BabyTokenizer:

  def encode(self, text):
    # Text → Numbers
    # "cat" → [13, 11, 29]
    return [self.char_to_idx[ch] for ch in text]

  def decode(self, indices):
    # Numbers → Text
    # [13, 11, 29] → "cat"
    return "".join([self.idx_to_char[i] for i in indices])

📁 File: step2_tokenizer.py

The tokenizer builds a lookup table: each unique character gets a unique number. That's the entire "vocabulary" of our model!

STEP 4

🎭 The Causal Mask

Rule: You can only look BACKWARDS, never at the future!

When predicting the next character, the model can only see previous characters. It's like covering future answers on a test!

Attention Mask for "the cat"

✅ = can see    ❌ = hidden (future)

🎭 Multi-Head Attention

Instead of ONE person searching, send 4 researchers (heads), each looking for different patterns. Then combine their findings! Our model uses 4 heads.

STEP 6

🔄 The Training Loop

4 lines that teach the model everything

for step in range(2000):

  # 1. FORWARD: Feed data, get predictions
  logits, loss = model(x_batch, y_batch)

  # 2. BACKWARD: Which weights caused mistakes?
  optimizer.zero_grad()  # Clear old gradients
  loss.backward()      # Compute new gradients

  # 3. UPDATE: Nudge weights to reduce loss
  optimizer.step()

➡ Forward Pass

Data flows through the model. Each layer processes it. Output = predictions.

⬅ Backward Pass

Error flows backwards. Each weight learns how much it contributed to mistakes.

🔄 Update

Optimizer adjusts weights slightly. AdamW is the "smart coach" that decides how much.
