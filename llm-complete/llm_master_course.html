4. Softmax & Temperature

# Softmax: raw scores → probabilities def softmax(logits, temperature=1.0): scaled = logits / temperature # temperature scaling exps = np.exp(scaled - np.max(scaled)) return exps / exps.sum() # Low temp → confident: [0.95, 0.03, 0.02] # High temp → spread: [0.40, 0.35, 0.25]

5. Self-Attention (Q, K, V)

Level 1 — Baby
🔍 Library Analogy: Query (Q) = your question, Key (K) = book titles on the shelf, Value (V) = book contents. You compare your question to each title (Q·K), find the best matches, then read those books (weighted sum of V). That's attention!
# Self-Attention: every token looks at every other token Q = x @ W_q # What am I looking for? K = x @ W_k # What do I contain? V = x @ W_v # What info do I have? scores = (Q @ K.T) / sqrt(d_k) # compatibility weights = softmax(scores) # normalize output = weights @ V # weighted sum

11. BPE Tokenizer

Level 2 — Teenage
📦 Compression Analogy: BPE (Byte-Pair Encoding) starts with individual characters, then repeatedly merges the most common pair. "t"+"h"→"th", "th"+"e"→"the". It's like ZIP compression for text — common patterns get shorter codes. GPT uses ~50,000 BPE tokens.
# BPE: merge most frequent pairs iteratively # Start: ['l','o','w','e','r'] → freq of 'lo'=5, 'ow'=3 # Merge 1: 'l'+'o' → 'lo' → ['lo','w','e','r'] # Merge 2: 'lo'+'w' → 'low' → ['low','e','r'] def get_most_frequent_pair(tokens): pairs = Counter(zip(tokens, tokens[1:])) return pairs.most_common(1)[0][0]

13. Dropout

Level 2 — Teenage
🎲 Study Group Analogy: Dropout randomly "turns off" neurons during training (like randomly removing students from a study group). This forces the remaining neurons to be more robust — no single neuron can be a lazy freeloader. At test time, everyone participates.

14. GELU Activation

Level 2 — Teenage
🚪 Smart Door Analogy: ReLU is a simple door: open (positive) or closed (negative). GELU is a smart door that opens gradually — slightly negative values can still peek through. This smooth transition helps gradients flow better and is used by GPT-2, BERT, and most modern LLMs.
# ReLU: max(0, x) — hard cutoff at 0 # GELU: x * Φ(x) — smooth, probabilistic gate # Φ(x) = cumulative distribution of normal dist. def gelu(x): return 0.5 * x * (1 + torch.tanh( math.sqrt(2/math.pi) * (x + 0.044715 * x**3)))

15. KV-Cache

Level 2 — Teenage
📝 Notes Analogy: Without KV-Cache, the model re-reads the entire conversation every time it generates one token. With KV-Cache, it keeps notes (cached Keys and Values) from previous tokens and only computes the new token's Q, K, V. Like taking notes in class instead of re-reading the textbook every question!
# Without cache: recompute K,V for ALL tokens each step # With cache: only compute K,V for the NEW token # Step 1: "The" → compute K,V for [The] # Step 2: "The cat" → reuse K,V for [The], compute for [cat] # Step 3: "The cat sat" → reuse [The,cat], compute [sat] # Speedup: O(n²) → O(n) per step! cached_k = torch.cat([cached_k, new_k], dim=1)

16. Learning Rate Scheduling

Level 2 — Teenage
🚗 Driving Analogy: Learning rate is your speed. Start slow (warmup — getting on the highway), go fast (peak learning), then slow down (cosine decay — approaching your destination). If you go too fast the whole time, you'll overshoot. Too slow, you'll never arrive.
# Cosine LR schedule with warmup def get_lr(step, warmup=100, max_steps=1000): if step < warmup: return max_lr * step / warmup # linear warmup progress = (step - warmup) / (max_steps - warmup) return min_lr + 0.5*(max_lr-min_lr)*( 1 + math.cos(math.pi * progress)) # cosine decay

17. Top-p (Nucleus) Sampling

Level 2 — Teenage
🎰 VIP List Analogy: Top-p sampling keeps the smallest set of tokens whose combined probability exceeds p (e.g., 0.9). If one token has 95% probability, only that one is considered. If probabilities are spread out, many tokens make the cut. It adapts automatically!
# Top-p: keep smallest set with cumulative prob ≥ p def top_p_sample(logits, p=0.9): sorted_probs, indices = torch.sort( softmax(logits), descending=True) cumsum = torch.cumsum(sorted_probs, dim=-1) mask = cumsum - sorted_probs > p # remove excess sorted_probs[mask] = 0 return torch.multinomial(sorted_probs, 1)

18. Perplexity

Level 2 — Teenage
🤔 Confusion Meter: Perplexity = 2^(cross-entropy loss). It measures how "confused" the model is. Perplexity of 10 means the model is as confused as if it had to randomly choose between 10 equally likely options. Lower = better. GPT-4 has perplexity ~5 on common text.
# Perplexity = e^(average cross-entropy loss) # Or equivalently: 2^(loss in bits) loss = F.cross_entropy(logits, targets) perplexity = torch.exp(loss) # PPL=1: perfect prediction (impossible) # PPL=10: choosing between ~10 options # PPL=50000: random guess over full vocab

20. RoPE (Rotary Position Embeddings)

Level 3 — Adult
🕐 Clock Analogy: Instead of a lookup table for positions (limited length!), RoPE rotates Q and K vectors by an angle proportional to their position. Position 0 = 12 o'clock, position 1 = rotated slightly. The angle BETWEEN two positions encodes their relative distance. Used by LLaMA, Mistral, and most modern LLMs.
# RoPE: rotate Q,K by position-dependent angle freqs = 1.0 / (10000 ** (torch.arange(0, dim, 2) / dim)) angles = positions.unsqueeze(1) * freqs.unsqueeze(0) # Apply 2D rotation to each dimension pair x_rot = x_even * cos(θ) - x_odd * sin(θ) y_rot = x_even * sin(θ) + x_odd * cos(θ)

21. Grouped-Query Attention (GQA)

Level 3 — Adult
🏢 Office Analogy: In standard MHA, every Q head has its own K and V heads (expensive!). In GQA, multiple Q heads share K,V heads — like employees sharing secretaries. MQA = everyone shares ONE secretary. GQA = groups of 2-4 share. Saves 50-75% KV-cache memory!

25. Beam Search

Level 3 — Adult
🌳 Path Analogy: Greedy search always takes the best-looking path at each fork. Beam search explores B paths simultaneously and picks the best OVERALL journey. Beam width=1 is greedy, width=5 explores 5 candidates. Used for translation and when you need deterministic, high-quality output.
# Beam search: keep top B candidates at each step beams = [(start_tokens, 0.0)] # (sequence, score) for step in range(max_len): candidates = [] for seq, score in beams: probs = model(seq)[-1] # next token probs for tok, p in top_k(probs, B): candidates.append((seq+[tok], score+log(p))) beams = sorted(candidates)[:B] # keep top B

26. RMSNorm

Level 3 — Adult
📏 Ruler Analogy: LayerNorm measures both center (mean) and spread (variance), then adjusts both. RMSNorm only measures the "size" (root mean square) and scales it — skipping the mean computation and the shift parameter. Simpler, ~10% faster, same quality. Used by LLaMA.

27. Mixed Precision (FP16/BF16)

30. Mixture of Experts (MoE)

Level 4 — Expert

# MoE: route each token to top-K experts class MoELayer(nn.Module): experts = [FFN() for _ in range(8)] # 8 experts router = nn.Linear(dim, 8) # gating network def forward(self, x): gates = softmax(self.router(x)) # which experts? top2 = gates.topk(2) # pick top 2 return sum(g * expert(x) for g, expert in top2)

31. LoRA (Low-Rank Adaptation)

Level 4 — Expert
🎨 Sticker Analogy: Instead of repainting an entire car (full fine-tuning), LoRA adds small stickers (low-rank matrices A and B). The original weights stay frozen, and you only train tiny adapter matrices. A 7B model needs ~14GB for full fine-tuning but only ~50MB for LoRA! Rank r=8 is common.
# LoRA: W_new = W_frozen + A × B # W: [4096 × 4096] = 16M params (frozen!) # A: [4096 × 8] = 32K params (trainable) # B: [8 × 4096] = 32K params (trainable) # Total trainable: 64K vs 16M = 0.4%! class LoRALinear(nn.Module): def forward(self, x): return self.frozen(x) + self.B(self.A(x)) * α/r

32. Quantization (INT4/INT8)

Level 4 — Expert
🗜️ JPEG Analogy: Like compressing a photo from RAW to JPEG — you lose some detail but the file is way smaller. Quantization converts FP16 weights (16 bits) to INT8 (8 bits) or INT4 (4 bits). A 7B model: FP16=14GB → INT8=7GB → INT4=3.5GB. Runs on consumer GPUs!
# Quantization: reduce precision of weights # FP16: 3.14159 → 16 bits # INT8: 3.14159 → 3 (8 bits, 256 levels) # INT4: 3.14159 → 3 (4 bits, 16 levels!) scale = (max_val - min_val) / (2**bits - 1) quantized = round((weight - min_val) / scale) # Dequantize: weight ≈ quantized * scale + min_val

34. Speculative Decoding

Level 4 — Expert
🏃 Assistant Analogy: A fast, small "draft" model guesses the next K tokens. Then the big "target" model verifies them all at once (in parallel!). If the draft is right (often 70-90% of the time), you get K tokens for the cost of 1. If wrong, you reject and regenerate. 2-3× speedup with identical output!
# Speculative Decoding # 1. Draft model generates K tokens quickly draft_tokens = small_model.generate(K=5) # 2. Target model verifies ALL at once target_probs = big_model(draft_tokens) # parallel! # 3. Accept tokens where draft matches target for i, tok in enumerate(draft_tokens): if random() < target_probs[i][tok] / draft_probs[i][tok]: accept(tok) # keep it! else: reject_and_resample() # fix it

35. DPO (Direct Preference Optimization)

Level 4 — Expert
🎯 Shortcut Analogy: RLHF needs 3 steps (SFT → Reward Model → PPO). DPO is a shortcut — it skips the reward model entirely! It directly optimizes the LLM using preference pairs (chosen vs rejected). Same result, simpler pipeline, more stable training. Used by LLaMA-3, Zephyr, and many open models.
# DPO: directly optimize from preferences # No reward model needed! # Loss = -log σ(β * (log π(y_w|x)/π_ref(y_w|x) # - log π(y_l|x)/π_ref(y_l|x))) # y_w = chosen (winner), y_l = rejected (loser) loss = -F.logsigmoid(β * ( log_ratio_chosen - log_ratio_rejected))

37. Hallucination & Safety

Safety measures include: RLHF alignment, constitutional AI, guardrails, output filtering, and RAG for grounding. No model is hallucination-free yet!
